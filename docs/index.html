⚙️ 三種執行模式 ⚙️ Execution Modes

模式ModeFlagCourtPlanVerify速度Speed
🚀 Quick--mode quick❌❌❌最快Fastest
⚖️ Hybrid--mode hybrid❌✅❌平衡Balanced
🛡️ Tight--mode tight（default）✅✅✅最全面Most thorough

Quick

純 LLM call，無 court、無 planning、無 verify。最快，適合簡單查詢。Pure LLM call — no court, no planning, no verify. Fastest. Best for simple queries.

Hybrid

有 step plan 引導執行，但 skip court 同 verify。平衡速度同可控性。Step plan guides execution, but skips court and verify. Balanced speed and control.

Tight（預設）

完整 pipeline：Plan → Devil 審查 → Angel 執行 → 每步 verify → checkpoint → report。最安全。Full pipeline: Plan → Devil review → Angel execute → per-step verify → checkpoint → report. Most secure.

# 改預設模式

# 單次 override

⚖️ 天使/魔鬼法庭 ⚖️ Angel/Devil Court

BAW 嘅核心機制。每次執行任務前，Devil 先發言挑戰，Angel 聽完先決定點做。 The heart of BAW. Before every task, the Devil speaks first as opposing counsel, then the Angel responds and decides.

流程 Flow

  1. 用戶落指令 — e.g.「delete all .log files」User gives command — e.g. "delete all .log files"
  2. 👿 Devil 先發言 — 指出風險，但無工具權限，得把口👿 Devil speaks first — identifies risks, but has ZERO execution power
  3. Devil 比風險分數 — [Devil: 7/10]Devil scores risk — [Devil: 7/10]
  4. 😇 Angel 聽完回應 — 有齊工具，可以執行😇 Angel responds — has full tool access, can execute
  5. 比對分數 — Angel 分數 > Devil → 繼續；否則 BLOCKCompare scores — Angel score > Devil → proceed; else BLOCK
  6. 執行 — Angel 用工具做嘢，逐 step checkpointExecute — Angel uses tools, step-by-step with checkpoints
$ baw "清空 /tmp 嘅 .tmp 檔案"

👿 Devil review: [Devil: 6/10]
  危險操作。雖然係 /tmp，但可能有正在使用嘅 temp file。
  建議先 list 出要 delete 嘅檔案，確認無 process 使用。

😇 Angel response: [Angel: 8/10]
  同意 Devil。我會先 list files → check lsof →
  只 delete 無使用嘅 .tmp files → 報告結果。

📋 Plan:
  1. ls -la /tmp/*.tmp 列出目標
  2. lsof /tmp/*.tmp 檢查使用中
  3. rm 無使用嘅檔案
  4. 報告共刪除 N 個檔案

可以用 baw --cfg set adversarial.enabled false 熄咗法庭。 Disable the court with baw --cfg set adversarial.enabled false.

💡 /rethink 用法Using /rethink

當你覺得 BAW 上次做得唔好，/rethink 會用同一個 prompt 但完全唔同嘅分析角度重做一次。 If BAW's last response wasn't great, /rethink re-runs the same prompt with a completely different analytical approach.

🔄 Scheduler 排程 🔄 Scheduler

BAW 內置 cron-based 定時任務系統： BAW has a built-in cron-based scheduled task system: 

# 每日凌晨 3 點做 backup
baw --schedule-add daily-backup '0 3 * * *' 'backup important files'

# 每小時 check disk
baw --schedule-add hourly-disk '0 * * * *' 'check disk > 90%'

# 列表
baw --schedule-list

# 移除
baw --schedule-rm hourly-disk

Scheduler 行 background daemon（每 60 秒 poll 一次），支援 cron 表達式、last-run 追蹤、避免 duplicate。 Background daemon polls every 60 seconds. Supports cron expressions, last-run tracking, and duplicate prevention.

核心設計原則 Core Design Principles

  • 永不問用戶Never Ask the User — BAW 自己 planning → executing → recovering，唔會拋問題返俾用戶。BAW plans, executes, and recovers on its own — never defers decisions to the user.
  • 永不放棄Never Surrender — 6 種策略用盡先上報。Tool timeout → 加倍 timeout → parent dir fallback → replan → rollback。Exhausts 6 strategies before reporting. Tool timeout → double timeout → parent dir fallback → replan → rollback.
  • 協議無關Protocol Agnostic — OpenAI / Anthropic / Google 任你揀，config.yaml 一行搞定。OpenAI / Anthropic / Google — one line in config.yaml.
  • 開放可插拔Open and Pluggable — Search provider、LLM protocol、tools 全部可以註冊擴展。Search providers, LLM protocols, and tools are all registerable and extensible.
