1 — Unified Memory（一層 API + 內部評分網絡）

關鍵 correction from the user：「一層夠，多層只係麻煩」。Agent 唔需要知 memory 分幾層。Agent 只需要 baw remember "X" 同 baw search "Y"。
內部嘅 hot/cold storage 係 implementation detail，唔 expose 俾 agent。

Agent 見到嘅 API（得呢兩個）：

$ baw remember "永遠用繁體中文，技術術語留英文"
→ 記低咗 (score: 0.50)

$ baw search "繁體"
→ 1. [score 0.92] HARD GATE: 永遠繁體中文...
   2. [score 0.78] HARD GATE: 唔好用簡體（相關聯 boosted）
   3. [score 0.12] 三個月前用過嘅某 API（低分，自動排底）

內部儲存（agent 唔需要知）：

~/.baw/memory/
├── store.jsonl       ← 所有記憶，一行一條 JSON
└── edges.json        ← 記憶間的相關聯網絡（可選）

// 每條記憶（agent 層面就咁睇到 content + score）
{
  "id": "mem_abc123",
  "content": "HARD GATE: 永遠繁體中文（技術術語留英文）",
  "type": "rule",
  "created": "2026-06-07T14:00:00Z",
  "last_accessed": "2026-06-07T16:30:00Z",
  "access_count": 47,
  "score": 0.92,
  "tags":["gate","language"]
}

評分機制（核心創新，來自 the user）：

事件分數變化
記憶被存取（search hit / session inject / 手動 recall）+0.05（direct boost）
直接相關記憶（edges 上的鄰居）+0.05 × edge_weight（associative spread）
二度相關記憶（鄰居的鄰居）+0.02 × edge_weight × 0.5（decaying）
24 小時冇存取−0.01（slow decay）
7 日冇存取−0.05/day（加速 decay）
Verify agent 評分 < 5−0.10（低品質）

用途： 高分記憶自動 inject 入 context、search 結果按 score 排序、低分記憶自動壓縮歸檔。好似人類記憶 — 成日提嘅嘢印象深刻，關聯記憶都一齊 boosted。

唔會有嘅問題（vs Hermes）：

  • ❌ 冇 State.db monolithic 黑盒 — 一行一條 JSONL，corrupt 都只係死一條
  • ❌ 冇 MEMORY.md drift guard — 記憶同 file system 完全分開
  • ❌ 冇 Vault git merge conflict — 記憶係 append-only JSONL，唔係多人改同一個 file

2 — Built-in Fact Checker（No Hallucination Architecture）

唔係一個 skill — 係 core 層面嘅 constraint。每次 agent 想做出 factual claim 時，自動 trigger：

Agent 想講「呢個 hosting plan 月費 $99」
  │
  ├─→ 係咪 fact? → 係 → 有 source 嗎？
  │     │                    │
  │     │               🟢 有 source link → allow
  │     │               🔴 冇 source → block + 自動 scrape
  │     │
  └─→ 係咪 opinion? → allow（唔使 source）

三種 fact-check mode：

Mode行為例子
strict所有 factual claim 必須有 source link / scrap result，否則 block報價、技術規格、歷史事件
normalFactual claim 盡量搵 source，冇 source 就注明「unsourced」日常對話
relaxed容許 common knowledge 唔使 source「Python 係 programming language」

Default mode: normal。當 task 涉及 price / spec / date 時自動升 strict。

3 — Model Router + Auto-fallback

Agent 唔需要知用邊個 model。Agent 只需要講想做乜，router 自動揀：

$ baw "幫我 summarize 呢篇文"
→ Router: task=summarization, 揀咗 MiniMax-M3（平 + 快 + 夠用）

$ baw "幫我睇下呢張圖有咩"
→ Router: task=vision, DeepSeek V4 Flash 冇 vision → auto-fallback MiniMax M3 vision

$ baw "寫個 production web app"
→ Router: task=code_complex, 揀咗 Claude Sonnet（貴但 code quality 好）

Router 嘅 decision matrix：

Task typeDefault modelFallback if fail
chat / simple QADeepSeek V4 Flash（平）MiniMax-M3
code generationMiniMax M3DeepSeek（慢但準）
vision / imageMiniMax M3（有 vision）fallback mmx CLI
long context (>32K)DeepSeek（1M context）MiniMax-M3
factual / researchDeepSeek（低 hallucination）Claude

One-click switch: baw --model claude-sonnet "do X" 直接 override，唔使改 config file。

4 — Tone Profiles（一鍵轉 tone）

唔靠 prompt 記 tone rule — 係 core tone layer，所有 output 必經：

Profile用嚟行為
business客戶文件、正式 email「合作邀約」代替「申請/懇請/請求授予」· 唔出個人名 · 唔出 deadline 壓力字眼
casual日常同 the user 吹水粵語口語 · 短句 · emoji OK
client-docClient facing 文件零 comment · 零 meta · 零個人名 · 直接出 artifact
teaching寫教學文件直接俾 .md file · 唔問「要唔要 PDF/v2/加章節」
ot-rt快速執行模式OT✅RT✅ — 做完先報，唔使逐 step confirm
stepwise逐步執行模式每步報 result，等 the user 確認先繼續

Switch: baw --tone business "寫 proposal" 或 session 中 /tone casual。

5 — Cost Transparency

每句 response 自動 show cost，唔使另整 skill：

$ baw "list files"
→ core/loop.py  core/llm.py  core/tools.py
  [tokens: 127 in · 45 out · $0.0003 | session total: $0.02]

Built-in 嘅 cost features：

  • Per-request meter — 每句 dialogue show token count + cost
  • Session total — 呢個 session 用咗幾多
  • Daily / monthly budget alert — threshold 到咗自動 warn
  • Model tier auto-selection — simple task → cheap model（上面 router 已做）

6 — Permission by Risk Level

唔係 binary approve/deny。係 risk-based：

Risk level行為例子
🔴 high一定要問 the user改 /etc/ · rm -rf · deploy production · 用新 API key
🟡 medium問一次，可以「always allow this session」write_file · bash · network access
🟢 low自動 allowread_file · web_search · 日常操作

呢個係 BAW 區別於 Hermes 嘅關鍵 — Hermes 要 create auto-allowlist skill 先做到，BAW 係 built-in。

7 — Graceful Tool Degradation

Tool fail 自動 fallback，唔係 error — 唔使 create fallback skill：

// Built-in fallback chain for each tool
vision_analyze:
  → MiniMax M3 vision
  → mmx vision CLI
  → "呢張圖我睇唔到，請描述"

web_search:
  → web_search API
  → curl fallback
  → "search 暫時 unavailable"

TTS:
  → MiniMax TTS
  → edge-tts
  → "TTS 暫時 unavailable"

每個 tool 有 health status — 知道邊個 work 邊個唔 work，唔使試完先知。

8 — Scheduler + Health Dashboard

Scheduler：

  • Cron syntax + interval + one-shot
  • Dependency chain — Job A 完咗 auto-trigger Job B
  • Path safety — 全部 absolute path，唔用 ~
  • Central output — baw job list 睇所有 job 嘅 status/output/error
  • Auto-retry — fail 咗自動 retry 3 次

9 — Async Delegation

Delegation 係 async，唔係 synchronous：

$ baw "研究下呢個 topic，做完話我知"
→ [Task #42 已放 background]
→ BAW 繼續做其他嘢
→ [Task #42 done] 研究結果：...

// 可以 check status
$ baw task status 42
→ In progress: 3/5 sources scraped, estimated 2 min remaining

// Subagent 可以返嚟問問題（唔係一 fail 就死）
$ baw task 42
→ Subagent: 「我見到兩個唔同嘅 price，邊個啱？」

Key improvements vs Hermes：

  • Background execution — parent 中斷唔會 kill subagent
  • Cost estimate before delegate — 「呢個 task 預計 $0.05，繼續？」
  • Subagent can ask questions — 唔係 black box fail
  • State inspection — 可以睇 subagent 做到邊

10 — Checkpoint + Verify（每步評分）

任何 state-changing op → auto checkpoint → spawn Verify agent（fresh, no memory, no prefix）→ 評分 0-10 → < 7 最多 loop 3 次。

Built-in — 唔係 skill，唔係外掛。

Main Agent 完成 step
  │
  ├─→ Auto save checkpoint（change log 寫一條）
  │
  └─→ Spawn Verify agent（fresh, no context）
        │
        ├─ Score ≥ 7 → ✅ → 繼續
        │
        └─ Score < 7 → ❌ → 修正 → 再 Verify（max 3 次）
                                │
                          3 次都 fail → 等 the user 決定
