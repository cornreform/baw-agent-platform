1️⃣ BAW 係咩？

BAW (Black And White) 係一套由零開始構建嘅 Agent Platform，唔依賴任何現有 framework。 佢嘅名嚟自兩隻狗仔（黑白配），代表系統嘅核心哲學：

  • 🤍 Angel — 執行者，有工具權限，負責規劃同執行
  • 🖤 Devil — 反對派，無任何工具權限，永遠先發言挑戰

BAW 嘅設計理念係 「永不放棄、永不問用戶」。 遇到問題時，BAW 會自己規劃步驟、嘗試替代方案、checkpoint rollback，所有策略用盡先上報。 佢唔會問你「呢個應該點做？」——佢自己做決定。

🎯 適合邊啲人用？

• 開發者/工程師 — 自動化 coding, debugging, deploy
• DevOps — 系統監控、排程任務、自動修復
• 資料分析 — 網頁搜索、記憶查詢、檔案處理
• 任何需要 靠得住、唔煩人 嘅 agent

2️⃣ 核心特色

⚖️

Angel/Devil 法庭

Devil 永遠先發言審查，Angel 聽完再執行。Devil 分數 > Angel 即 STOP。

🧠

永不放棄

失敗 → retry → replan → rollback。6 種策略用盡先上報，唔煩人。

🔌

協議無關 LLM

OpenAI / Anthropic / Google 協議通吃。一行 config 轉模型。

💰

透明成本

每次 LLM call 顯示 per-token cost。預算控制。

📝

自我學習技能

俾句描述 / URL，BAW 自動分析、拆解、生成 YAML skill。

📊

HTML Dashboard

一鍵生成深色主題 dashboard：系統狀態、排程、技能、活動。

🎭

6 種語氣 Profile

casual / business / teaching / client-doc / ot-rt / stepwise — 隨時切換。

🔍

開放 Search Provider

內置 DuckDuckGo（無需 Key），可升級 Tavily/Brave/Firecrawl。

🔄

Scheduler 排程

Cron 表達式定時執行。背景 daemon 60s 輪詢。

📁

檔案版本歷史

每次寫入 ISO timestamp + SHA256 manifest + 自動 git commit。

🛡️

三級權限

High（禁止 sudo/rm -rf）/ Medium（提示）/ Low（允許）。

✅

Per-step Verify

每步 LLM 驗證結果，FAIL 自動 recover（retry / replan / rollback）。

🔧

Tool Degradation

bash timeout → 加倍 → parent directory fallback。write_file 失敗 → /tmp/。

🎮

互動式 Chat

彩色 terminal MU*, Tab 補全 slash commands。單句或聊天模式。

⚙️

Setup Wizard

互動引導：mode → tone → court → fact check → model。即時生效。

💡

BTW 快捷模式

baw --btw "問題" — 快速 LLM 一問一答，無 court 無 plan。

🏗️

Async Delegation

baw --delegate "task" — 背景執行，主 terminal 即刻 free。

3️⃣ 安裝 & 設定

最低要求

  • Python 3.11+（建議 3.12）
  • macOS / Linux
  • Git（auto-commit 用）

安裝步驟

# 1. Clone repo
git clone https://github.com/cornreform/baw-agent-platform.git ~/baw
cd ~/baw

# 2. 安裝 dependencies（用 uv 或 pip）
pip install pyyaml # 核心依賴
pip install duckduckgo-search # 內置 web search

# 3. 建立 CLI wrapper
ln -sf ~/baw/baw ~/.local/bin/baw

基本 Config

BAW 嘅 config 喺 ~/.baw/config.yaml。 第一次用 baw 會自動用 repo 入面嘅預設 config。 

# ~/.baw/config.yaml 結構
model:
  default: "deepseek-v4-flash"          # 主要模型
  fallback: "deepseek-v4-flash"         # fallback 模型

providers:
  deepseek:                             # 每個 provider 一個 section
    base_url: "https://api.deepseek.com/v1"
    api_key_env: "DEEPSEEK_API_KEY"     # 從環境變數讀 key
    models:
      - id: "deepseek-v4-flash"
        context_window: 65536
        cost_per_1m_input: 0.30
        cost_per_1m_output: 1.20

mode: "tight"                          # 執行模式
tone:
  default: "casual"                    # 語氣
adversarial:
  enabled: true                        # 開法庭
fact_check:
  mode: "normal"                       # 事實查證強度

API Keys

將 API keys 放入 ~/.baw/.env 或 shell 環境變數： 

# ~/.baw/.env
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
MINIMAX_API_KEY=mc-xxxxxxxxxxxxxxxx

7️⃣ 三種執行模式

BAW 有三種執行模式，平衡速度同完整度：

模式用法CourtPlanVerify速度
🚀 Quick--mode quick❌❌❌最快
⚖️ Hybrid--mode hybrid❌✅❌平衡
🛡️ Tight--mode tight（預設）✅✅✅最全面

Quick Mode

純粹 LLM call，無 court 無 planning。最快但無安全審查。適合簡單查詢、BTW 模式。

Hybrid Mode

有 step plan 引導執行，但 skip 咗 court 同 verify。平衡速度同可控性。

Tight Mode（預設）

完整 pipeline：Plan → Devil 審查 → Angel 執行 → 每步 verify → checkpoint。最安全、最全面。

可以在 config 設定預設 mode，或用 --mode flag override： 

# 默認 tight，有時 quick
baw --cfg set mode hybrid       # 改默認
baw --mode quick "而家幾點？" # 單次 override

8️⃣ 天使/魔鬼法庭

法庭係 BAW 嘅核心。每次執行任務前，Devil 會先發言，Angel 聽完先決定點做。

流程

  1. 用戶落指令 — 「delete all .log files」
  2. 👿 Devil 先發言 — 「咁做可能會 delete 錯重要 log...」
    Devil 無任何工具權限，得把口。
  3. Devil 比風險分數 — [Devil: 7/10]
  4. 😇 Angel 聽完回應 — 「OK，我先 check 邊啲 log 係舊嘅，唔 delete active logs」
    Angel 有齊工具，可以執行。
  5. 比對分數 — Angel 分數 > Devil 分數 → 繼續執行；否則 BLOCK
  6. 執行 — Angel 用工具做嘢，每步 checkpoint
baw "清空 /tmp 嘅 .tmp 檔案"

👿 Devil review: [Devil: 6/10]
  危險操作。雖然係 /tmp，但可能有正在使用嘅 temp file。
  建議先 list 出要 delete 嘅檔案，確認無 process 使用。
  可以用 lsof 檢查。

😇 Angel response: [Angel: 8/10]
  同意 Devil。我會先 list files → check lsof →
  只 delete 無使用嘅 .tmp files → 報告結果。

📋 Plan:
  1. ls -la /tmp/*.tmp 列出目標
  2. lsof /tmp/*.tmp 檢查使用中
  3. rm 無使用嘅檔案
  4. 報告共刪除 N 個檔案

▶️ Step 1: 列出 /tmp/*.tmp ...
✅ Done — 4/4 steps
刪除咗 12 個 .tmp 檔案，有 2 個被 process 使用中已保留。

法庭可以用 baw --cfg set adversarial.enabled false 熄咗佢。 喺 chat 中可以用 /court 睇返上次審判記錄。

1️⃣1️⃣ Scheduler 排程

BAW 內置 cron-based scheduler，可以定時自動執行任務： 

# 新增排程：每日凌晨 3 點做 backup
baw --schedule-add daily-backup '0 3 * * *' 'backup important files'

# 每小時檢查 disk
baw --schedule-add hourly-disk '0 * * * *' 'check disk usage and alert if > 90%'

# 列表、移除
baw --schedule-list
baw --schedule-rm hourly-disk

Scheduler 行 background daemon（每 60 秒 poll 一次），唔會阻住你個 terminal。 支援 cron 表達式、last-run 追蹤、避免 duplicate。

1️⃣3️⃣ 背景任務 Async Delegation

長時間任務可以 delegate 去背景執行，主 terminal 即刻 free： 

# Delegate 去背景（主 terminal 即時可用）
baw --delegate "分析呢個 project 嘅 code quality"
📋 Task ID: baw_20260607_123456_abc

# Check 狀態（任何時候）
baw --task-id baw_20260607_123456_abc
📋 Task: baw_20260607_123456_abc
   Status: running
   Goal: 分析呢個 project 嘅 code quality

# 全部任務一覧
baw --tasks

# 取消
baw --task-cancel baw_20260607_123456_abc

TaskManager 會自動管理 concurrency（最多 3 個同時行），超出嘅會自動排隊。

1️⃣8️⃣ 記憶系統

BAW 嘅記憶係 JSONL append‑only + 內部評分機制： 

# 記低偏好
baw --remember "Always use Traditional Chinese for responses"
Remembered (score: 1.00)

# 搜尋記憶
baw --search chinese
[0.92] Always use Traditional Chinese for responses

# Chat 模式
BAW> /remember 語氣用 casual
BAW> /search 語氣

記憶系統會隨使用自動評分：重複存取 → 加分。相關記憶 → 聯想擴散。 記憶會注入每次 agent run 嘅 system prompt。

1️⃣9️⃣ 事實查證

BAW 內置三級事實查證：

Mode行為
off唔檢查，最快
normal（預設）標記可疑宣稱
strict阻擋無法驗證嘅宣稱

Normal mode 下，BAW 會標記「聽落唔對路」嘅陳述： 

baw "話我知 Raspberry Pi 5 嘅 specs"
⚠️ Fact Check: "8GB RAM" — 需要確認實際型號
⚠️ Fact Check: "2.4GHz quad-core" — 搜索結果唔一致
→ BAW 已自動用 web_search 驗證

Strict mode 下，無法驗證嘅宣稱會直接 block。

2️⃣0️⃣ 權限引擎

BAW 有三級風險權限，防止意外破壞：

風險包含行為
🔴 High/etc/*, rm -rf, sudo, dd, .env, *.key⛔ 禁止
🟡 Mediumwrite_file, bash (general)⚠️ 提示確認
🟢 Lowread_file, web_search✅ 允許

權限規則喺 config.yaml 嘅 permissions.risk_levels section， 可以用 path pattern 同 command prefix 定義。 

# config.yaml 權限配置示例
permissions:
  risk_levels:
    high:

    low:
      - tool: "read_file"        # 讀檔案直接俾
