# BAW 日常對話工作能力測試 V5
#使用說明：將以下消息逐條傳送給 Telegram Bot，每條獨立發送（用來測 session 記憶）
# 全部用自然廣東話，不含 "Task"、"執行"、"step" 等命令詞

---

## 測試組 A：閒聊 vs 工作 分界

### A1 — 純閒聊（應走 Chat Bypass，1-2 秒回覆）
```
今日天氣幾好嗎，你有出門呀？
```
**預期**：快速回覆，唔使 tool，唔顯示 progress

---

### A2 — 偽閒聊，實際要記憶（應走 Work Mode）
```
我頭先同你講過我鍾意食拉麵，你記得咩？
```
**預期**：用 memory search 工具查找，返回正確結果

---

### A3 — 自然請求，無命令詞
```
幫我記低，我下周三要去見客戶
```
**預期**：用 memory tool add，確認儲存成功

---

### A4 — 連續對話，上下文繼承（全部發送完才分析）
```
我叫 Sunny，做緊一個 AI project
```
**等待回覆後**，再發：
```
我個 project 要整合 Telegram bot
```
**等待回覆後**，再發：
```之前講過個 project，而家我遇到個 bug
```
**等待回覆後**，再發：
```
我個 bot 叫咩名？
```
**等待回覆後**，再發：
```
我係邊個？
```
**預期**：第五輪應答 "Sunny"。若第四輪回答 BAW/不記得，檢查 session memory 是否正常。

---

## 測試組 B：創意 + 資訊混合

### B1 — 需要即時資訊（須坦承限制）
```
而家香港有咩新戲上緊？我想睇戲。
```
**預期**：bot 坦承 "我唔能即時上網搜尋"（除非有 search tool），唔應亂講

---

### B2 — 創意寫作（純聊天）
```
寫首廣東話詩俾我，關於夏天同海邊
```
**預期**：走 Chat Bypass，直接輸出詩歌，唔使 tool

---

### B3 — 創意 + 檔案輸出
```幫我想個餐廳名，要粵語諧音喻，然後寫落 /tmp/restaurant-name.txt
```
**預期**：先諧音創意（LLM 推理），再寫檔案（tool），確認 /tmp/restaurant-name.txt 存在

## 測試組 C：陷阱與邊界

| 結果 | 說明 |
|------|------|
| ✅ Pass | 完全符合預期 |
